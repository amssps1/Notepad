Fase 1:
Instalar o SQL 2019 (Medusa/Minotauro); Novo Storage(?)
Instalar o SQL 2019 e redimensionar as VM's Priapo e EOS para suportarem todas as BDs

Fase 2:
Always ON - 
Utilizar o AG existente no Artemis/Perseu, adicionar um Listener ao AG (ex.) "LTArtemis" , "LTPerseu"  ;
Passar todas as BDs Locais para o AG ; 
no Always On replicar todas as BDs para o Priapo / EOS ;
Alterar Conection Strings das aplicações para redirecionarem os pedidos para o Listener ; 
Capturar os pedidos que ainda vem directo ao Artemis e Perseu;
Alterar connection strings até tudo não existir mais pedidos directos;
Passado 2/3 semanas, alterar no DNS os aliases "Artemis/Perseu" e reencaminhar para os Listeners;

Fase 3 (DR):
Retirar Node 2 de cada Cluster do WSFC;
Adicionar o Storage necessário para Replica a cada um destes Nos
Instalar e criar instância de SQL 2019
Restore de BDs
Adicionar Instancia ao AG

Failover de AG com interrupção efetivamente a zero !!!!! (Esperamos :) ).

Dúvidas:
a) Fase 1 pode passar para última fase para não existir interrupção de serviço ?
